# Core Rush 게임 기획서

## 1. 한 줄 정의

`Core Rush`는 플레이어가 직접 공격하지 않아도 자동으로 전투가 진행되는 모바일 액션 로그라이크 디펜스 게임이다. 플레이어는 주로 이동, 위치 선정, 웨이브 사이의 로그라이크 선택지를 통해 공격 방식, 이동 방식, 코어 방어 방식을 바꾼다.

## 2. 기획 의도

기존 카드 오토배틀러 방향은 선택의 재미는 있었지만, 사용자가 원한 액션 RPG 감각과는 거리가 있었다. 이 게임은 `탕탕특공대`, `궁수의 전설`, 캐주얼 디펜스 게임의 장점을 섞되, 전투 조작 부담을 낮추고 “내 선택 때문에 전투 양상이 바뀐다”는 로그라이크 감각을 중심에 둔다.

목표 감각:

- 조작은 가볍다.
- 전투는 계속 자동으로 벌어진다.
- 선택지는 단순 스탯업이 아니라 전투 방식 자체를 바꾼다.
- 적이 코어를 향해 몰려오므로 자연스럽게 디펜스 목표가 생긴다.
- 한 판은 짧고, 여러 빌드를 빠르게 실험할 수 있다.

## 3. 핵심 플레이어 경험

플레이어는 화면 중앙 근처의 코어를 지키는 영웅이다. 적들은 사방에서 몰려오고, 영웅과 코어는 자동으로 적을 공격한다. 영웅은 기본적으로 코어 주변을 자동으로 움직이며 적을 피하고, 플레이어가 화면을 드래그하면 직접 이동에 개입할 수 있다.

전투 중 적을 처치하면 경험치를 얻고, 레벨업 시 선택지가 등장한다. 선택지는 카드가 아니라 `모듈`, `스킬`, `전술 강화`처럼 표현한다. 선택한 강화에 따라 이후 자동 전투의 모양이 바뀐다.

예시:

- `쌍열 발사`: 자동 사격이 1발에서 2발로 바뀐다.
- `관통 화살`: 탄환이 적을 뚫고 지나간다.
- `자동 대시`: 이동 중 주기적으로 돌진하며 적을 밀친다.
- `플라즈마 잔상`: 이동 경로에 피해 장판이 생긴다.
- `코어 포탑`: 코어가 추가 자동 사격을 한다.
- `냉기장`: 영웅 주변 적을 둔화시킨다.

## 4. 핵심 루프

1. 전투 시작
2. 영웅이 자동 이동/자동 공격
3. 적 웨이브가 코어로 접근
4. 영웅과 코어가 적을 처치
5. 경험치 획득
6. 레벨업 또는 웨이브 클리어 시 증강 보드에서 6개 선택지 중 1개 선택
7. 공격/이동/방어 방식 변화
8. 더 강한 웨이브 등장
9. 코어가 파괴되거나 제한 시간을 버티면 런 종료

## 5. 조작 원칙

기본 조작은 “선택과 개입”이다.

- 영웅 이동은 기본 자동.
- 플레이어는 드래그로 이동 방향에 개입할 수 있다.
- 공격은 완전 자동.
- 스킬 사용 버튼은 기본적으로 만들지 않는다.
- 런 중 능동 선택은 레벨업 선택지에 집중한다.

이 방향은 모바일 한 손 플레이에 맞고, `탕탕특공대`류의 이동 집중형 플레이보다 더 캐주얼하다.

## 6. 전투 구조

### 코어

코어는 방어 목표다.

- 코어 HP가 0이 되면 실패.
- 코어는 기본 포탑을 갖고 자동으로 적을 공격한다.
- 일부 강화는 코어의 방어 능력을 키운다.
- 코어가 있으면 플레이어가 단순 생존만 하지 않고 “위치 방어”를 고민하게 된다.

### 영웅

영웅은 자동 전투의 중심이다.

- 자동으로 코어 주변을 돌며 적과 거리를 유지한다.
- 드래그 입력 시 수동 이동이 우선된다.
- 기본 공격은 가장 가까운 적을 자동 조준한다.
- 이동 강화 선택에 따라 움직임이 공격 수단이 된다.

### 적

적은 단순하지만 역할이 달라야 한다.

- 일반 적: 코어로 천천히 접근.
- 러너: 빠르게 접근하지만 체력이 낮음.
- 브루트: 느리지만 체력이 높고 코어 피해가 큼.
- 보스: 특정 웨이브마다 등장하는 강한 적.

초기 프로토타입에서는 적 종류를 명확히 보여주는 색/크기 차이만 있어도 충분하다.

## 7. 로그라이크 선택지 설계

선택지는 매 레벨업 또는 주요 웨이브 클리어 시 `증강 보드` 형태로 제공한다. 기본안은 6개 중 1개 선택이다. 각 선택지는 “숫자 증가”보다 “전투 방식 변화”와 “보유 선택지와의 시너지”를 우선한다.

선택지 간 시너지의 세부 설계는 별도 문서인 [로그라이크 선택지 시너지 설계](choice-synergy-design.md)를 기준으로 한다.

선택지 카테고리:

- 공격 방식
- 이동 방식
- 코어 방어
- 범위 제어
- 성장/회복

### 공격 방식

- `속사 모듈`: 공격 간격 감소
- `고압 탄환`: 탄환 피해 증가
- `쌍열 발사`: 탄환 수 증가
- `관통 화살`: 탄환 관통
- `수호 궤도검`: 영웅 주변 회전 공격

### 이동 방식

- `기동 부츠`: 기본 이동 속도 증가
- `자동 대시`: 이동 중 주기적 돌진 및 넉백
- `플라즈마 잔상`: 이동 경로에 피해 장판 생성
- `충격 파동`: 이동 중 주변 적 밀쳐냄

### 코어 방어

- `코어 포탑`: 코어 자동 사격 추가
- `긴급 수리`: 코어/영웅 회복 및 최대 HP 증가
- `냉기장`: 영웅 주변 둔화장

### 성장 보조

- `경험치 자석`: 경험치 획득 반경 증가

## 8. 빌드 예시

### 잔상 돌격자

핵심 선택:

- 자동 대시
- 플라즈마 잔상
- 충격 파동

플레이 감각:

영웅이 코어 주변을 빠르게 돌며 지나간 자리에 피해 장판을 남긴다. 이동 자체가 공격이 된다.

### 빙결 포탑 수비대

핵심 선택:

- 코어 포탑
- 냉기장
- 긴급 수리

플레이 감각:

코어 주변을 요새화하고, 적을 느리게 만들어 포탑으로 녹인다. 디펜스 게임 느낌이 강하다.

### 관통 저격수

핵심 선택:

- 고압 탄환
- 관통 화살
- 속사 모듈

플레이 감각:

몰려오는 적을 한 줄로 뚫어내는 탄막 액션 느낌이 강하다.

### 근접 탄막 레인저

핵심 선택:

- 수호 궤도검
- 쌍열 발사
- 기동 부츠

플레이 감각:

영웅 주변을 도는 무기와 다중 탄환으로 근접 압박을 정리한다.

## 9. 런 구조

초기 목표는 3분 런이다.

- 0:00-0:30: 기본 조작과 자동 전투 이해
- 0:30-1:30: 첫 빌드 방향 형성
- 1:30-2:30: 적 밀도 증가, 빌드 효과 체감
- 2:30-3:00: 보스 또는 대형 웨이브

프로토타입에서는 3분 생존을 승리 조건으로 둔다.

향후 모바일 캐주얼 버전에서는 5분 런까지 확장할 수 있다.

## 10. 화면 구성

### 전투 화면

필수 요소:

- 코어 HP
- 영웅 HP
- 웨이브
- 레벨
- 시간
- 처치 수
- 경험치 바
- 캔버스 전장

모바일 기준:

- 전투 화면이 가장 커야 한다.
- 버튼은 최소화한다.
- 레벨업 선택지는 6개 증강 보드로 표시한다.
- 조이스틱은 보조 입력이므로 과하게 강조하지 않는다.

### 레벨업 화면

레벨업 시 게임은 잠시 멈춘다. 선택지는 카드가 아니라 모듈 버튼처럼 표현한다.

각 선택지에는 다음 정보를 표시한다.

- 이름
- 아이콘 한 글자
- 짧은 효과 설명
- 태그
- 현재 보유 선택지와의 시너지 이유

## 12. 다음 개발 우선순위

### 1순위: 게임성 검증

- 초반 60초가 지루하지 않은지
- 자동 이동이 답답하지 않은지
- 수동 개입이 의미 있는지
- 레벨업 선택지가 실제로 전투 양상을 바꾸는지
- 코어 방어 목표가 부담이 아니라 긴장감으로 느껴지는지

### 2순위: 선택지 차별화

현재 선택지는 기능적으로 다르지만 시각적 피드백이 약할 수 있다.

추가하면 좋은 피드백:

- 대시 잔상
- 장판 색상
- 관통 탄환 효과
- 포탑 발사 이펙트
- 냉기장 링
- 레벨업 선택 후 짧은 연출

### 3순위: 적 패턴

추가 후보:

- 원거리 적
- 자폭 적
- 코어만 노리는 적
- 플레이어를 추적하는 암살자
- 보호막 적

### 4순위: 런 결과

런 종료 후 보여줄 정보:

- 생존 시간
- 처치 수
- 최종 빌드명
- 선택한 강화 목록
- 받은 코어 피해

## 13. 피해야 할 방향

- 카드 게임처럼 보이는 UI로 돌아가지 않는다.
- 전투 중 스킬 버튼을 너무 많이 만들지 않는다.
- 선택지를 단순 `공격력 +10%` 위주로 만들지 않는다.
- 플레이어가 아무것도 안 하면 바로 죽는 밸런스는 피한다.
- 자동 이동이 너무 완벽해서 플레이어 개입이 필요 없어지는 것도 피한다.

## 14. 미해결 질문

- 플레이어가 이동을 직접 조작하는 비중은 어느 정도가 좋은가?
- 코어 방어가 핵심인가, 영웅 생존이 핵심인가?
- 한 판 길이는 3분이 좋은가, 5분이 좋은가?
- 증강 보드는 6개가 적당한가, 5개 또는 8개가 더 좋은가?
- 빌드 조합은 4개 정도면 충분한가, 더 많은 조합 축이 필요한가?

## 15. 다음 세션 인수인계 메모

다른 세션에서 이어받을 때는 먼저 이 문서를 읽고, 현재 구현이 문서의 방향과 맞는지 확인한다.

추천 작업 순서:

1. 모바일에서 현재 배포 URL을 직접 플레이한다.
2. 초반 30초의 자동 이동/자동 전투 감각을 평가한다.
3. 레벨업 선택지가 전투 방식 변화를 충분히 주는지 확인한다.
4. 부족하면 `app.js`의 `upgradePool`, `getAutoMoveVector`, `updateMovementTech`를 먼저 수정한다.
5. 이후에 적 패턴과 이펙트를 늘린다.
