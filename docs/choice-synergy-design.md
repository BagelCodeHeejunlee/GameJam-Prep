# 로그라이크 선택지 시너지 설계

## 1. 방향 전환

이 게임의 로그라이크 선택지는 선형 트리 방식이 아니다.

원하는 감각은 `A를 골랐으니 다음엔 B, 그 다음엔 C`가 아니라, 롤체 증강이나 롤 아레나 증강처럼 여러 선택지가 펼쳐지고, 플레이어가 현재 상황과 보유 조합을 보고 그 안에서 시너지를 읽어내는 방식이다.

핵심 문장:

> 선택지는 길을 따라가는 트리가 아니라, 매번 펼쳐지는 증강 보드 안에서 조합을 발견하는 재료다.

## 2. 참고작에서 가져올 것

### 롤체 증강

롤체의 증강은 정해진 트리를 타는 것이 아니라, 특정 타이밍에 여러 선택지 중 하나를 고르며 현재 보드/아이템/특성/경제 상태와 맞춰 판단한다. Riot의 TFT 패치 노트에서도 증강은 전투 증강, 경제 증강, 특성 증강처럼 다양한 역할로 밸런싱된다. 최근 패치에서도 econ Augment와 combat Augment를 구분해 조정하고 있다.

이 게임에 적용할 점:

- 선택지는 한 줄 트리가 아니라 역할이 다른 증강들의 묶음이다.
- 선택지는 현재 빌드 상태에 따라 가치가 달라져야 한다.
- 같은 선택지도 어떤 조합에서는 핵심이고, 다른 조합에서는 애매해야 한다.
- 일부 선택지는 당장 강하지 않아도 이후 선택지의 가치를 바꿔야 한다.

### 롤 아레나 증강

롤 아레나 증강은 챔피언의 원래 플레이 패턴을 비트는 방식이 강하다. Riot의 Arena 개발 글에서도 새 증강을 추가하면서 “재미있고 화려한 선택지”와 증강 레벨 같은 변형을 강조한다.

이 게임에 적용할 점:

- 선택지는 단순 수치업보다 플레이 패턴을 바꿔야 한다.
- 공격 방식, 이동 방식, 방어 방식이 선택지로 변형되어야 한다.
- 선택지는 발견하는 재미가 있어야 한다.

### Challengers!

Challengers!에서 가져올 것은 트리가 아니라 `덱이 특정 성격으로 굳어지는 느낌`이다. 여러 선택을 통해 내 런의 색깔이 자연스럽게 만들어져야 한다.

### Slay the Spire

Slay the Spire에서 가져올 것은 `현재 내 조합 때문에 선택지의 가치가 계속 바뀌는 구조`다. 같은 선택지도 지금 가진 조건과 보상 장치에 따라 다르게 평가되어야 한다.

### Hearthstone Arena

Hearthstone Arena에서 가져올 것은 `3개 후보 안에서 지금 강한 것과 앞으로 좋아질 것을 판단하는 드래프트 감각`이다.

## 3. 선택지 화면 구조

방 클리어 또는 주요 보상 타이밍에 `증강 보드`를 연다.

기본안:

- 모든 보드는 한 번에 3개 선택지 표시
- 그중 1개 선택
- 중간보스/희귀 보상도 후보 수는 3개로 고정
- 큰 보상은 3지선다 2회 선택 또는 희귀도 가중치 증가로 표현
- 1회 무료 리롤 가능
- 선택지에는 태그와 현재 시너지 이유를 표시
- 희귀도는 노말, 레어, 에픽, 전설 4단계로 구분

선택지에는 두 종류의 수량이 있다.

- 보드 후보 수: 한 번 열렸을 때 화면에 보이는 선택지 개수
- 스테이지 선택 횟수: 한 스테이지 동안 실제로 고르는 횟수

스테이지 선택 횟수 기준:

- 최소 보장: 10회
- 권장 목표: 12-14회
- 중간보스/이벤트/상점 보너스 포함: 15회 이상 가능

보드 후보 수는 3개로 고정한다. 보상을 더 크게 만들고 싶을 때도 후보 수를 늘리지 않고 선택 횟수, 리롤, 희귀도, 보상 품질을 조정한다.

- 3개: 모바일에서 읽기 쉽고 선택 부담이 낮음
- 4개 이상: 정보량이 늘면서 자동 전투 감상 흐름을 끊을 수 있음

초기 기준은 `모든 보드 3개 중 1개 선택`이다.

## 4. 증강 보드 구성 규칙

3개 선택지는 완전 무작위가 아니라 아래 역할 비율로 구성한다.

| 슬롯 | 역할 | 의도 |
| --- | --- | --- |
| 1 | 현재 보유 태그/트리거와 직접 시너지 | 지금 빌드를 이어갈 선택 |
| 2 | 새 빌드 축 또는 하이브리드 선택 | 방향 전환 또는 조합 확장 |
| 3 | 생존/경제/희귀 선택 | 리스크 관리 또는 발견의 재미 |

중간보스/희귀 보드도 후보 수는 3개로 유지한다. 대신 아래 중 하나로 보상감을 준다.

- 3개 후보의 희귀도 가중치를 올린다.
- 3지선다 보드를 2회 연속 제공한다.
- 3개 후보 중 최소 2개를 현재 빌드의 상위 강화로 구성한다.

이 구조의 목적은 “정답 하나”를 주는 것이 아니라, 플레이어가 자신의 런을 해석하게 만드는 것이다.

예:

현재 보유:

- 플라즈마 잔상
- 자동 대시
- 기동 부츠

증강 보드:

- `장판 확장`: 플라즈마 잔상과 직접 시너지
- `대시 지뢰`: 자동 대시와 장판 피해를 연결
- `감전 탄환`: 새 상태이상 축

여기서 정답은 하나가 아니다.

- 장판 빌드를 밀면 `장판 확장`
- 대시+장판 하이브리드를 원하면 `대시 지뢰`
- 새 빌드를 섞고 싶으면 `감전 탄환`

## 5. 선택지 풀 구조

선택지는 `기본 선택지 풀`과 `캐릭터 선택지 풀`로 나눈다.

- 기본 선택지 풀: 모든 캐릭터가 공통으로 사용할 수 있는 선택지
- 캐릭터 선택지 풀: 선택한 캐릭터의 자동 이동 성향, 고유 패시브, 추천 무기와 강하게 연결되는 선택지

현재 런의 전체 선택지 풀은 아래처럼 만든다.

```text
현재 선택지 풀 = 유저 레벨로 해금된 기본 선택지 + 선택 캐릭터의 캐릭터 레벨로 해금된 캐릭터 선택지
```

기준:

- 유저 레벨이 높을수록 더 많은 기본 선택지와 높은 등급 기본 선택지가 해금된다.
- 캐릭터 레벨이 높을수록 해당 캐릭터 전용 선택지와 높은 등급 캐릭터 선택지가 해금된다.
- 선택지 등급은 `노말`, `레어`, `에픽`, `전설` 4단계다.
- 보상 후보 수는 등급과 관계없이 3개로 고정한다.
- 높은 등급 보상은 후보 수 증가가 아니라 등장 가중치, 추가 선택권, 보상 품질로 표현한다.

기본 선택지 풀의 구체 목록은 [기본 선택지 풀](base-augment-pool.md)을 기준으로 한다.

## 6. 시너지의 기본 단위

선형 트리 대신 아래 네 가지 단위를 사용한다.

### 1. 태그

선택지의 계열을 뜻한다.

기본 태그:

- `탄환`
- `이동`
- `장판`
- `상태`
- `처치`
- `경제`
- `방어`
- `표식`

태그는 “내가 어떤 쪽으로 기울고 있는지”를 보여준다.

조건부 태그:

- `소환`
- `설치`
- `코어`

조건부 태그는 모든 캐릭터에게 열지 않는다. 소환/설치/코어 전용 선택지는 해당 기능을 가진 캐릭터, 무기, 스테이지 규칙이 있을 때만 전체 선택지 풀에 들어간다.

### 2. 트리거

효과가 발동하는 조건이다.

기본 트리거:

- `공격할 때`
- `대시할 때`
- `이동할 때`
- `적 처치 시`
- `적이 상태이상일 때`
- `전술 데이터 획득 시`
- `피해를 받을 때`
- `방 시작 시`

트리거가 같으면 서로 붙기 쉽다.

예:

- `자동 대시`: 대시 트리거를 만든다.
- `대시 지뢰`: 대시할 때 지뢰를 남긴다.
- `플라즈마 잔상`: 이동할 때 장판을 남긴다.

### 3. 출력

선택지가 만들어내는 결과다.

출력 예:

- 피해
- 둔화
- 폭발
- 장판
- 회복
- 보호막
- 전술 데이터
- 끌어당김
- 밀쳐냄

출력이 다른 선택지의 입력이 되면 시너지가 생긴다.

예:

- `감전 탄환`의 출력은 `감전 상태`
- `번개 폭발`의 입력은 `감전 상태 적 사망`

### 4. 변환

한 자원을 다른 자원으로 바꾸는 규칙이다.

예:

- 회복량을 공격력으로 변환
- 전술 데이터 획득을 광역 피해로 변환
- 보호막 획득량을 주변 피해로 변환
- 이동 거리를 장판 지속시간으로 변환

변환 선택지는 하이브리드 조합을 만든다.

## 7. 선택지 유형

기존의 `시동기 → 증폭기 → 완성기` 표현은 너무 선형적이므로 폐기한다. 대신 선택지를 다음 유형으로 분류한다.

### 축 생성형

새로운 행동이나 오브젝트를 만든다.

예:

- 플라즈마 잔상
- 감전 탄환
- 자동 대시
- 냉기장

### 조건 활용형

이미 존재하는 조건을 활용한다.

예:

- 장판 위 적 처치 시 폭발
- 감전 적 처치 시 연쇄 피해
- 표식 적 처치 시 공격 속도 증가
- 둔화된 적 처치 시 냉기 폭발

### 연결형

두 축을 이어준다.

예:

- 대시할 때 지뢰 설치
- 장판 위 적을 중력장 중심으로 끌어당김
- 표식 적 처치 시 탄환 효과 재발동
- 전술 데이터 획득 시 다음 방 시작 폭발

### 증폭형

이미 가진 축의 효율을 높인다.

예:

- 탄환 수 증가
- 장판 지속시간 증가
- 상태이상 지속시간 증가
- 표식 피해 증가

### 안전형

런을 안정화한다.

예:

- 긴급 회복
- 대시 보호막
- 적 둔화
- 피해 감소

### 경제형

미래 선택을 더 좋게 만든다.

예:

- 데이터 자석
- 리롤 장치
- 다음 증강 선택 1회 추가
- 특정 태그 등장 확률 증가

## 8. 비선형 시너지 패턴

### A. 같은 트리거 공유

여러 선택지가 같은 트리거에 붙는다.

예: `대시할 때`

- 자동 대시
- 대시 지뢰
- 대시 보호막
- 무한 궤도

플레이어는 대시 빌드를 “정해진 순서”로 타는 것이 아니라, 보드에 나온 대시 관련 증강을 보고 조합한다.

### B. 출력과 입력 연결

한 선택지의 출력이 다른 선택지의 입력이 된다.

예:

- 감전 탄환: 적에게 감전 부여
- 감전 전이: 감전 피해가 주변 적에게 전파
- 번개 폭발: 감전 적 사망 시 주변 피해

순서는 고정되지 않아도 된다. `감전 전이`를 먼저 고르면 당장은 약하지만, 이후 감전 부여 선택지를 찾고 싶어진다.

### C. 역할 보완

서로 다른 약점을 메운다.

예:

- 관통 탄환은 줄지은 적에게 강함
- 냉기장은 적을 느리게 만들어 줄 세우기 쉬움
- 중력장은 적을 한 지점에 모음

세 선택지는 직접 텍스트로 연결되어 있지 않아도 플레이 감각상 시너지가 난다.

### D. 자원 변환

한 빌드의 부산물을 다른 빌드의 연료로 바꾼다.

예:

- 데이터 자석 + 데이터 연소
- 긴급 회복 + 과충전 보호막
- 표식 사냥 + 탄막 과부하

### E. 공간 시너지

액션 디펜스이므로 공간 자체도 시너지 대상이다.

예:

- 장판은 지역을 만든다.
- 중력장은 적을 모은다.
- 대시는 이동 경로를 만든다.
- 냉기장은 안전한 간격을 만든다.

공간형 시너지는 카드 게임보다 이 게임에 더 잘 맞는다.

### F. 선택지 풀 조작

선택지가 미래 보드 자체를 바꾼다.

예:

- 다음 보드에 `이동` 태그 2개 보장
- 현재 가장 많은 태그의 선택지 등장률 증가
- 현재 보드 후보 중 1개 잠금
- 리롤 시 같은 태그 유지

이 방식은 롤체식 증강 판단과 잘 맞는다.

## 9. 증강 보드 예시

### 예시 1: 초반 무방향

보유 선택지 없음.

보드:

- 플라즈마 잔상
- 충격 파동
- 감전 탄환

의도:

초반에는 어느 축으로 갈지 고르는 단계다. 이때는 시너지보다 방향 선택이 중요하다.

### 예시 2: 이동 장판 쪽으로 기운 상태

보유:

- 플라즈마 잔상
- 자동 대시

보드:

- 장판 확장
- 대시 지뢰
- 감전 탄환

가능한 판단:

- `장판 확장`: 현재 빌드 직접 강화
- `대시 지뢰`: 대시 트리거를 장판/폭발 축과 연결
- `감전 탄환`: 새 상태이상 축

### 예시 3: 방어/생존 쪽으로 기운 상태

보유:

- 긴급 회복
- 대시 보호막

보드:

- 과충전 보호막
- 충격 파동
- 데이터 자석

가능한 판단:

- `과충전 보호막`: 회복/보호막을 공격으로 변환
- `충격 파동`: 접근한 적을 밀어내 안정화
- `데이터 자석`: 이후 보상 품질을 노리는 경제 선택

### 예시 4: 상태이상 쪽으로 기운 상태

보유:

- 감전 탄환
- 냉기장

보드:

- 감전 전이
- 번개 폭발
- 폭풍 전도체

가능한 판단:

- `감전 전이`: 상태 확산
- `번개 폭발`: 처치 보상
- `폭풍 전도체`: 상태이상 빌드의 전설급 확장

## 11. 프로토타입 기본 선택지 풀

프로토타입의 기본 선택지 풀은 총 33개로 시작한다.

- 노말 10개
- 레어 10개
- 에픽 8개
- 전설 5개

구체 목록, 해금 레벨, 태그, 효과, 대표 시너지는 [기본 선택지 풀](base-augment-pool.md)에 정리한다.

캐릭터 선택지 풀은 다음 단계에서 각 캐릭터별로 추가한다. 캐릭터 선택지는 공통 선택지보다 캐릭터의 자동 행동 방식과 고유 패시브를 더 직접적으로 바꿔야 한다.

## 12. 증강 보드 생성 알고리즘

증강 보드는 아래 순서로 만든다.

1. 현재 보유 선택지에서 태그 카운트 계산
2. 현재 보유 선택지에서 트리거 목록 계산
3. 직접 시너지 후보군 생성
4. 트리거 공유 후보군 생성
5. 새 축 후보군 생성
6. 생존/경제 후보군 생성
7. 희귀 선택지 후보군 생성
8. 중복을 제거하고 3개를 표시

의사 코드:

```js
function generateAugmentBoard(state) {
  const pool = buildUnlockedPool({
    userLevel: state.userLevel,
    character: state.character,
    characterLevel: state.characterLevel
  });
  const tags = countTags(state.augments);
  const triggers = collectTriggers(state.augments);
  const rewardType = state.rewardType;

  return [
    pickDirectSynergy(pool, tags, triggers, rewardType),
    pickBridgeOrPivot(pool, tags, triggers, rewardType),
    pickUtilityOrWildcard(pool, tags, rewardType)
  ];
}
```

중요한 점:

- 특정 조합을 강제로 완성시키지 않는다.
- 현재 조합을 읽을 수 있는 선택지를 최소 1개 제공한다.
- 새 방향 선택지를 항상 1개 이상 제공한다.
- 생존 선택지를 너무 자주 강제하지 않는다.
- 고희귀 선택지는 강력하지만 빌드 방향을 흐릴 수도 있어야 한다.
- 한 스테이지에서 증강 선택 기회는 최소 10회 보장한다.
- 중간보스 보상은 후보 수 증가가 아니라 추가 선택권 또는 희귀도 증가로 체감시킨다.

## 13. 선택지 UI 표시 규칙

각 선택지는 아래 정보를 보여준다.

- 이름
- 태그 1-2개
- 효과 설명
- 현재 시너지 이유
- 희귀도

예:

```text
대시 지뢰
태그: 이동 / 장판
효과: 대시한 지점에 지연 폭발 지뢰 설치
시너지: 자동 대시 보유, 장판 확장 보유
```

시너지 이유는 꼭 보여주는 것이 좋다. 이 정보가 없으면 플레이어는 선택지가 왜 좋은지 알기 어렵다.

## 14. 구현 우선순위

### 1순위

- 선택지 데이터에 `tags`, `triggers`, `inputs`, `outputs`, `rarity` 추가
- 유저 레벨 기반 기본 선택지 해금 구현
- 캐릭터 레벨 기반 캐릭터 선택지 해금 구현
- 모든 보상 3개 후보 구조 추가
- 중간보스/희귀 보상의 추가 선택권 또는 희귀도 가중치 추가
- 스테이지당 최소 10회 이상 증강 선택 구조 추가
- 각 선택지에 “현재 시너지 이유” 표시

### 2순위

- 증강 보드 생성 알고리즘 구현
- 리롤 1회 추가
- 선택지 잠금 기능 검토

### 3순위

- 공간 시너지 강화
- 상태이상 시너지 강화
- 조건부 코어/소환 캐릭터 전용 선택지 추가

## 15. 피해야 할 것

- 선택지를 선형 트리처럼 만들지 않는다.
- `장판 1 → 장판 2 → 장판 3` 같은 강제 루트를 만들지 않는다.
- 특정 조합을 맞추지 못하면 망하는 구조를 피한다.
- 모든 선택지를 현재 빌드와만 맞춰주지 않는다. 새 방향의 유혹이 있어야 한다.
- 단순 수치업만 늘리지 않는다.

## 16. 최종 기준

좋은 선택지 보드는 플레이어가 이렇게 고민하게 만든다.

- “지금은 장판을 밀 수 있는데, 대시 지뢰로 폭발 축을 섞을까?”
- “감전이 하나도 없지만 번개 폭발이 너무 좋아 보이는데 먼저 집어둘까?”
- “이번 판은 회복과 보호막이 많으니 과충전 보호막을 노려볼까?”
- “이 선택은 당장은 약한데 다음 보드가 좋아질 것 같다.”

이 고민이 생기면 성공이다.
