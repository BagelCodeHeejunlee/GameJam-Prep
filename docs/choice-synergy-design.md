# 로그라이크 선택지 시너지 설계

## 1. 핵심 관점

이 게임에서 선택지는 단독으로 강한 능력이 아니라, 다음 선택의 가치를 바꾸는 장치여야 한다.

좋은 선택지는 다음 질문을 만든다.

- 이걸 골랐으니 다음엔 무엇을 찾고 싶어지는가?
- 이 선택 때문에 원래 애매했던 선택지가 좋아지는가?
- 내 빌드가 특정 방향으로 기울었다는 느낌이 생기는가?
- 2개 조합, 3개 조합, 완성 조합이 단계적으로 보이는가?

기준 문장:

> 선택지는 능력이 아니라 빌드의 문맥을 만드는 부품이다.

## 2. 참고작에서 가져올 철학

### Challengers!에서 가져올 것

Challengers!의 핵심은 전투 조작보다 `덱 구성`, `덱 압축`, `카드 조합`, `라운드 사이 선택`에 있다. 중요한 감각은 “지금 넣은 카드 때문에 다음 라운드의 덱 구조가 달라진다”는 점이다.

이 게임에 적용할 점:

- 선택지가 내 빌드 풀을 특정 방향으로 기울인다.
- 같은 계열을 쌓으면 임계점 보상이 열린다.
- 덱 압축처럼, 선택지도 무조건 많이 갖는 것이 아니라 핵심 축을 선명하게 만드는 것이 중요하다.
- 자동 전투이기 때문에 전투 전 선택의 책임감이 커야 한다.

### Slay the Spire에서 가져올 것

Slay the Spire의 강점은 `카드`, `유물`, `상태`, `비용`, `드로우`, `제거`가 서로 얽히는 방식이다. 카드 하나의 절대 가치보다 현재 덱과 유물에 맞는지가 중요하다.

이 게임에 적용할 점:

- 선택지는 `시동기`, `증폭기`, `보상기`, `완성기`로 나눈다.
- 초반 선택은 방향을 열고, 중반 선택은 그 방향을 증폭하고, 후반 선택은 완성한다.
- 단순 피해 증가보다 “조건을 만들고, 그 조건을 소비하고, 소비했을 때 보상을 받는 구조”가 좋다.

### Hearthstone Arena에서 가져올 것

Hearthstone Arena의 재미는 제한된 선택지에서 덱의 곡선, 부족한 역할, 시너지 가능성을 계속 판단하는 데 있다. 매 선택은 지금 당장 강한 카드와 앞으로 강해질 카드 사이의 판단이 된다.

이 게임에 적용할 점:

- 매 레벨업 선택지는 3개만 제공한다.
- 선택지 3개는 서로 다른 역할이어야 한다.
- 현재 빌드와 맞는 선택지가 최소 1개는 있어야 한다.
- 가끔은 “지금은 약하지만 다음 선택을 열어주는 선택지”가 있어야 한다.

## 3. 선택지 역할 분류

선택지는 아래 역할 중 하나 이상을 가진다.

### 1. 시동기

빌드의 방향을 처음 여는 선택지.

예:

- `플라즈마 잔상`: 이동 경로에 피해 장판 생성
- `코어 포탑`: 코어가 자동 공격
- `감전 탄환`: 탄환이 감전 상태 부여
- `소환 드론`: 작은 드론 1기 소환

시동기는 단독으로도 작동해야 하지만, 완성형은 아니어야 한다.

### 2. 증폭기

이미 가진 시동기의 가치를 키우는 선택지.

예:

- `장판 확장`: 장판 반경 증가
- `포탑 속사`: 코어 포탑 발사 간격 감소
- `감전 전이`: 감전된 적 사망 시 주변에 감전 전파
- `드론 편대`: 드론 수 증가

증폭기는 관련 시동기가 없으면 애매해야 한다.

### 3. 보상기

특정 조건을 만족했을 때 추가 보상을 주는 선택지.

예:

- `장판 처형`: 장판 위에서 죽은 적이 폭발
- `포탑 과열`: 코어 포탑이 5회 공격할 때마다 광역탄 발사
- `감전 수확`: 감전된 적 처치 시 경험치 추가
- `드론 회수`: 드론이 처치한 적마다 코어 회복

보상기는 플레이어가 “내 빌드가 돌아간다”고 느끼게 만든다.

### 4. 연결기

서로 다른 빌드 축을 이어주는 선택지.

예:

- `자기장 장판`: 장판 위의 적을 코어 쪽으로 끌어당김
- `대시 충전`: 대시할 때 포탑이 즉시 1회 발사
- `드론 전도체`: 드론 공격이 감전을 부여
- `냉기 탄환`: 탄환 빌드와 상태이상 빌드를 연결

연결기는 하이브리드 빌드를 가능하게 한다.

### 5. 완성기

특정 조건을 만족해야 등장하는 강한 선택지.

예:

- `플라즈마 고속도로`: 장판 3개 이상 보유 시 등장. 장판이 서로 연결되어 지속 피해 증가.
- `요새화 코어`: 코어 관련 선택지 3개 이상 보유 시 등장. 코어 주변에 방어막 링 생성.
- `번개 폭풍`: 감전 관련 선택지 3개 이상 보유 시 등장. 감전 적이 주기적으로 연쇄 피해.
- `드론 항모`: 소환 관련 선택지 3개 이상 보유 시 등장. 코어가 드론 생산.

완성기는 플레이어가 빌드를 완성했다는 강한 신호가 된다.

## 4. 태그 시스템

모든 선택지는 태그를 가진다. 태그는 선택지 제공 확률과 완성기 해금에 사용한다.

기본 태그:

- `탄환`: 자동 사격, 관통, 다중 발사
- `이동`: 대시, 속도, 회피, 충격파
- `장판`: 잔상, 불, 전기, 독, 냉기 지역
- `코어`: 포탑, 방어막, 수리, 코어 반격
- `상태`: 감전, 냉기, 화상, 표식
- `소환`: 드론, 지뢰, 위성, 임시 유닛
- `처치`: 폭발, 연쇄, 경험치, 회복
- `경제`: 경험치, 리롤, 선택지 품질

각 선택지는 보통 1-2개의 태그를 가진다.

예:

- `플라즈마 잔상`: `이동`, `장판`
- `충격 파동`: `이동`, `처치`
- `코어 포탑`: `코어`, `탄환`
- `감전 탄환`: `탄환`, `상태`
- `드론 전도체`: `소환`, `상태`

## 5. 선택지 제공 규칙

선택지 시너지를 만들려면 완전 무작위만으로는 부족하다. 제공 규칙이 필요하다.

### 기본 규칙

레벨업 선택지 3개는 다음 구조를 따른다.

- 1개: 현재 빌드와 직접 시너지가 있는 선택지
- 1개: 새로운 빌드 방향을 여는 선택지
- 1개: 생존/경제/범용 선택지

이렇게 하면 플레이어가 항상 “이어가기”, “갈아타기”, “안전하게 가기” 중 하나를 고르게 된다.

### 시너지 가중치

선택지를 고르면 해당 태그의 선택지 등장 확률이 오른다.

예:

- `플라즈마 잔상` 선택 후: `장판`, `이동` 선택지 확률 증가
- `코어 포탑` 선택 후: `코어`, `탄환` 선택지 확률 증가
- `감전 탄환` 선택 후: `상태`, `탄환` 선택지 확률 증가

### 고립 방지

시동기를 골랐는데 이후 관련 선택지가 전혀 안 나오면 빌드가 끊긴다. 그래서 시동기 선택 후 2번의 레벨업 안에는 관련 증폭기나 보상기를 최소 1번 보여준다.

예:

- `플라즈마 잔상`을 골랐다면 다음 2회 안에 `장판 확장`, `장판 처형`, `플라즈마 고속도로 후보` 중 하나를 제공한다.

### 완성기 해금

같은 태그 선택지를 일정 수 이상 고르면 완성기가 선택지 풀에 들어간다.

예:

- `장판` 3개: `플라즈마 고속도로` 해금
- `코어` 3개: `요새화 코어` 해금
- `상태` 3개: `번개 폭풍` 해금
- `소환` 3개: `드론 항모` 해금

완성기는 무조건 제공하지 않고, 해금 후 선택지 풀에 들어간다.

## 6. 시너지 방식 목록

### A. 태그 누적 시너지

같은 태그를 여러 개 고르면 강한 선택지가 열린다.

예:

- `장판` 1개: 장판 생성
- `장판` 2개: 장판 반경/지속시간 강화
- `장판` 3개: 장판 완성기 해금

장점:

- 이해하기 쉽다.
- 빌드 방향이 명확하다.
- Challengers!식 “같은 방향으로 덱이 굳어지는 느낌”을 만들기 좋다.

### B. 시동기 + 보상기

먼저 조건을 만들고, 나중에 그 조건을 보상으로 바꾼다.

예:

- `감전 탄환`: 적에게 감전 부여
- `감전 수확`: 감전 적 처치 시 경험치 추가
- `번개 폭발`: 감전 적 사망 시 연쇄 피해

장점:

- Slay the Spire식 콤보 감각이 강하다.
- 첫 선택은 미래 투자, 두 번째 선택은 보상이라는 흐름이 생긴다.

### C. 변환 시너지

한 자원을 다른 자원으로 바꾼다.

예:

- `과충전 코어`: 코어 HP 회복량 일부를 포탑 피해로 전환
- `피해 흡수`: 영웅이 받은 피해 일부를 코어 보호막으로 전환
- `경험치 연소`: 경험치 획득 시 주변 적에게 작은 피해

장점:

- 평범한 선택지도 특정 빌드에서 갑자기 강해진다.
- 하이브리드 빌드를 만들기 좋다.

### D. 트리거 공유 시너지

서로 다른 선택지가 같은 트리거를 공유한다.

예:

- 트리거: `대시할 때`
- `자동 대시`: 주기적 대시 발생
- `대시 지뢰`: 대시 종료 위치에 지뢰 설치
- `대시 충전`: 대시할 때 코어 포탑 즉시 발사
- `대시 장벽`: 대시 경로에 보호막 생성

장점:

- 하나의 행동이 여러 효과를 동시에 일으키며 빌드가 돌아가는 느낌을 준다.

### E. 역할 보완 시너지

각 선택지가 서로 약점을 보완한다.

예:

- `관통 화살`: 줄지어 오는 적에게 강함
- `냉기장`: 적을 줄지어 모이게 만듦
- `자기장 장판`: 적을 한 지점으로 모음

장점:

- 직접적인 “A가 B를 강화”가 아니어도 플레이 감각상 시너지가 생긴다.

### F. 선택지 풀 조작

선택지가 이후 선택지의 품질이나 종류를 바꾼다.

예:

- `전술 스캐너`: 다음 레벨업 선택지 1개가 현재 빌드 태그와 반드시 일치
- `리롤 장치`: 매 선택마다 1회 리롤 가능
- `전문화`: 가장 많이 가진 태그의 선택지 등장률 증가, 다른 태그 등장률 감소

장점:

- Hearthstone Arena식 드래프트 판단을 강화한다.
- 선택이 단순 능력 획득이 아니라 미래 선택 관리가 된다.

### G. 중복 강화 시너지

같은 선택지를 여러 번 고르면 효과가 단순 증가하지 않고 형태가 바뀐다.

예:

- `코어 포탑 I`: 포탑 1개
- `코어 포탑 II`: 포탑 2개
- `코어 포탑 III`: 포탑이 회전 레이저로 변형

장점:

- 같은 선택을 반복하는 재미가 생긴다.
- 선택지 이름을 기억하기 쉽다.

## 7. 구체 선택지 조합표

| 먼저 고른 선택지 | 이후 고르면 좋은 선택지 | 시너지 결과 |
| --- | --- | --- |
| 플라즈마 잔상 | 장판 확장 | 이동 경로 장판이 넓어져 코어 주변 방어선이 됨 |
| 플라즈마 잔상 | 자동 대시 | 대시 경로 전체가 피해 장판으로 변함 |
| 플라즈마 잔상 | 장판 처형 | 장판 위에서 죽은 적이 폭발 |
| 자동 대시 | 충격 파동 | 대시할 때 광역 넉백과 피해가 동시에 발생 |
| 자동 대시 | 대시 충전 | 대시할 때 코어 포탑이 즉시 발사 |
| 충격 파동 | 냉기장 | 밀려난 적이 둔화되어 다시 접근하기 어려움 |
| 코어 포탑 | 포탑 속사 | 코어의 자동 방어 능력 강화 |
| 코어 포탑 | 대시 충전 | 영웅 이동이 코어 공격 트리거가 됨 |
| 코어 포탑 | 요새화 코어 | 코어 주변 방어막 링 생성 |
| 감전 탄환 | 감전 전이 | 감전된 적 사망 시 주변 적도 감전 |
| 감전 탄환 | 번개 폭발 | 감전 상태가 광역 피해 엔진이 됨 |
| 감전 탄환 | 드론 전도체 | 드론 공격도 감전을 부여 |
| 수호 궤도검 | 기동 부츠 | 빠르게 움직이며 근접 적을 베어냄 |
| 수호 궤도검 | 충격 파동 | 접근한 적을 베고 밀쳐내는 근접 방어 빌드 |
| 드론 소환 | 드론 편대 | 드론 수 증가 |
| 드론 소환 | 드론 회수 | 드론 처치가 코어 회복으로 전환 |
| 드론 소환 | 드론 전도체 | 소환 빌드가 상태이상 빌드와 연결 |
| 경험치 자석 | 전술 스캐너 | 레벨업 속도와 선택지 품질을 함께 올림 |
| 경험치 자석 | 경험치 연소 | 경험치 획득이 주변 피해로 변함 |
| 긴급 수리 | 과충전 코어 | 회복량 일부가 공격력으로 전환 |

- 코어 포탑
- 감전 탄환
- 수호 궤도검

## 10. 선택지 UI 원칙

선택지는 카드처럼 보이면 안 된다. 이 게임은 액션 디펜스이므로 `모듈`, `전술`, `개조 부품`처럼 보여야 한다.

표시 정보:

- 이름
- 짧은 설명
- 태그 1-2개
- 현재 빌드와의 연결 표시

연결 표시 예:

- `현재 보유: 플라즈마 잔상과 시너지`
- `장판 태그 2/3`
- `선택 시 플라즈마 고속도로 해금까지 1`
- `코어 포탑을 즉시 발동시키는 트리거`

## 11. 구현 규칙

`app.js`에서 선택지를 정의할 때 각 선택지는 최소한 다음 정보를 가진다.

```js
{
  id: "plasma_trail",
  name: "플라즈마 잔상",
  tags: ["movement", "field"],
  role: "starter",
  unlocks: ["field_expand", "field_execute", "plasma_highway"],
  synergies: ["auto_dash", "shockwave"],
  apply: (state) => {}
}
```

선택지 제공 알고리즘은 다음 흐름을 따른다.

1. 현재 보유 태그 카운트 계산
2. 현재 빌드와 직접 연결되는 선택지 후보 생성
3. 새 방향을 여는 시동기 후보 생성
4. 생존/경제 후보 생성
5. 각 그룹에서 1개씩 뽑아 3개 선택지 제공
6. 해금 조건을 만족한 완성기는 낮은 확률로 후보에 포함
