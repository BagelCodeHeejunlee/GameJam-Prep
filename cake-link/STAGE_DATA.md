# 케이크 링크 1~17 스테이지 데이터

실제 설정의 기준 파일은 `stages.js`다. 아래 표와 목록은 기획 검토용 요약이다.

## 표기 규칙

- 색: 딸기=`berry`, 레몬=`lemon`, 말차=`matcha`, 초코=`choco`, 블루베리=`blueberry`
- 보드: `1`은 사용할 수 있는 칸, `0`은 막힌 칸이다. 왼쪽 위부터 0~15 인덱스를 사용한다.
- 판 패턴: `a2b1`은 서로 다른 두 색이 2조각, 1조각 들어간 판이다. `a`, `b`, `c`의 실제 색은 색상 웨이트로 중복 없이 뽑는다.
- 첫 하단 3판은 `openingRack`으로 고정하고, 그다음부터 두 웨이트를 이용해 생성한다.
- `rules.capacity`는 한 판의 완성 조각 수이며 기본값은 6이다.
- 8~17은 `gimmick`으로 대표 기믹을 하나씩 지정한다. 세부 설정은 `mechanics`, 시작 접시의 메타데이터, `openingModifiers`에 저장한다.

일반 플레이어 모델로 각 스테이지를 10,000회 실행한 기준 클리어율은 100.00%, 96.16%, 95.44%, 89.78%, 87.14%, 85.50%, 80.00%다. 이 수치는 절대적인 사람 클리어율이 아니라 스테이지끼리 난이도를 비교하기 위한 기준값이다. 1~7은 잠긴 칸이나 특수 보드 없이 모두 기본 4×4 보드를 사용하며, 새로운 피처와 보드 제약은 8스테이지부터 도입한다.

8~17의 목표 범위는 `planner / expert` 프로필 100회 기준 70~90%다. 현재 고정 시드 100회 결과는 순서대로 79%, 89%, 83%, 89%, 82%, 85%, 79%, 86%, 76%, 80%다. 사람의 실제 클리어율이 아니라 스테이지 간 상대 난이도를 조정하는 자동 플레이 기준값이다.

### 스테이지 8~17 기믹 설정

- 8 색깔 접시: 5번 딸기2, 10번 레몬2 접시에는 지정된 색 조각만 올라갈 수 있다.
- 9 얼음 판: 5번과 10번 칸은 인접 판이 사라진 뒤 한 겹이 깨져 사용 가능해진다.
- 10 개구리: 딸기 2판을 완성하면서 `1111 / 0111 / 0111 / 0111` 보드의 0번 개구리를 15번 목표까지 이동한다. 판 이동 처리가 끝날 때마다 상하좌우 빈칸 중 최단 경로가 짧아지는 칸으로 한 칸 움직인다.
- 11 미스테리: 첫 하단 3판에 각각 한 조각이 포함되며 이후 판은 30% 확률로 1~2개가 포함된다. 실제 색은 생성 시 결정하지만 놓을 때까지 공개하지 않는다.
- 12 8조각: 모든 판의 용량과 완성 조건이 8이며 `a4`, `a3b1`, `a2b2` 중심으로 공급한다.
- 13 열쇠·잠금: 첫 딸기 완성은 5·9번, 첫 레몬 완성은 6·10번 칸을 연다.
- 14 순서 목표: `딸기 → 레몬 → 말차 → 딸기 → 레몬 → 말차 → 딸기` 순서다. 다른 색 완성은 실패가 아니라 순서 진행에서 제외한다.
- 15 무지개: 첫 하단은 `딸기4+무지개2`, `레몬5+무지개1`, `말차4+무지개1`이다. 이후 판은 24% 확률로 1~2개를 포함한다. 일반 조각 완성을 먼저 처리한 후 정확한 부족분에만 자동 사용한다.
- 16 깨지는 받침대: 중앙 5·6·9·10번 칸은 그 칸에서 케이크가 완성되면 사용할 수 없는 칸으로 바뀐다.
- 17 이층 케이크: 5번은 위 딸기3/아래 레몬3, 10번은 위 말차3/아래 초코3이다. 위층은 받기 전용이며 완성 후 아래층이 활성화된다.

## 데이터 구조

```js
{
  gimmick: { id: "ice", label: "얼음 판", icon: "❄️" },
  rules: { capacity: 6 },
  objective: { type: "counts" }, // 생략 시 색깔별 goals
  mechanics: {
    iceCells: [{ index: 5, layers: 1 }]
  },
  openingRack: [{ berry: 3 }, { lemon: 3 }, { matcha: 3 }],
  openingModifiers: [{}, {}, {}]
}
```

`openingRack`은 기존 편집기와 데이터 호환을 위해 일반 색 조각 맵을 유지한다. 미스테리 실제 색과 무지개 수는 같은 인덱스의 `openingModifiers`에 둔다. 런타임 판은 `createPlateData`가 `{pieces, capacity, hiddenColors, layers, allowedColor, receiveOnly}` 형태로 정규화한다.

## 공통 처리 순서

한 번의 배치는 다음 순서를 사용하며 실제 게임과 시뮬레이터가 같은 처리 함수를 공유한다.

1. 선택한 판의 미스테리 조각 공개
2. 일반 같은 색 정렬과 일반 조각 완성
3. 무지개 부족분 배정과 추가 완성
4. 완성·빈 판 제거와 이층 아래층 공개
5. 얼음, 열쇠, 깨지는 받침대 상태 변경
6. 개구리 한 칸 이동
7. 목표 달성과 남은 배치·가득 찬 보드 실패 확인

장기 밸런싱에서는 목표색이 일정 횟수 동안 나오지 않으면 다음 판에 보정하는 `pity` 규칙을 검토할 수 있다. 현재는 재현 가능한 `seed`, 초반 `openingRack`, 색·패턴·특수 조각 웨이트로 난이도를 조절한다.
