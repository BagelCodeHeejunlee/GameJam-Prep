# 3인 로테이션 로그라이크 타워 디펜스 기획서

작성일: 2026-07-04

## 1. 기획 목적

기존 게임과 별개로 새 폴더에서 독립적으로 진행하는 신규 게임 기획이다. 기존 루트 게임, 기존 `docs`, 기존 프로토타입 폴더의 파일은 건드리지 않고, 이 폴더 안에서만 기획과 이후 프로토타입 작업을 진행한다.

장르 방향:

- 로그라이크
- RPG
- 타워 디펜스
- 모바일 터치 기반 실시간 방어
- 모바일 세로형 게임

핵심 콘셉트:

> 중앙 타워를 향해 사방에서 몰려오는 적을, 최대 3명의 캐릭터가 3개의 방어 포지션을 빠르게 돌며 막아내는 로테이션 디펜스 RPG.

## 2. 핵심 재미

이 게임의 중심 재미는 복잡한 조작보다 **짧은 판단으로 전장을 재배치하는 재미**다.

- 적은 여러 방향에서 동시에 접근한다.
- 캐릭터는 각자 현재 맡은 방향과 포지션을 방어한다.
- 유저가 화면을 한 번 터치하면 모든 캐릭터가 시계방향 옆 포지션으로 이동한다.
- 어느 타이밍에 로테이션할지에 따라 방어 성공 여부가 갈린다.
- 적 처치와 전투 진행으로 경험치를 모으고, 레벨업할 때 로그라이크식 업그레이드를 골라 캐릭터의 공격 범위, 속성, 시너지, 담당 능력을 강화한다.

## 3. 기본 화면과 맵 구조

### 맵

- 타워는 맵 한가운데에 있다.
- 전장은 사각 맵이다.
- 적은 사각 맵의 모든 가장자리 또는 여러 지점에서 중앙 타워를 향해 몰려온다.
- 전장을 원형으로 만들거나 120도 구역선으로 명확하게 나누지 않는다.
- `콩콩던전`처럼 맵 전체에서 적이 몰려오는 감각을 기준으로 한다.
- 캐릭터는 타워 중앙을 기준으로 3개 조준 방향을 시계방향으로 돈다.
- 첫 프로토타입에서는 캐릭터가 타워 바깥 원형 궤도를 이동하지 않고, 중앙에서 공격 방향만 바뀐다.
- 임시 포지션명:
  - A 포지션
  - B 포지션
  - C 포지션

### 적 이동

- 적은 맵 바깥쪽에서 등장한다.
- 적은 각자의 등장 방향에서 중앙 타워를 향해 이동한다.
- 적이 타워에 닿으면 타워 체력이 감소한다.
- 타워 체력이 0이 되면 패배한다.

### 포지션과 공격 담당

- 맵을 눈에 보이는 3구역으로 구분하지 않는다.
- 캐릭터는 현재 조준 방향을 기준으로 캐릭터 고유의 사거리, 공격각, 스킬 규칙에 따라 적을 공격한다.
- 포지션은 캐릭터 배치와 로테이션을 위한 기준일 뿐, 맵을 강하게 쪼개는 UI 요소가 아니다.
- 첫 프로토타입에서 포지션은 실제 위치가 아니라 조준 방향이다.
- 포지션 자체에는 기본 공격 방향이나 기본 공격각이 없다.
- 공격 범위, 사거리, 탄 퍼짐, 광역 효과, 업그레이드에 따라 인접 포지션 방향이나 넓은 범위까지 타격할 수 있다.

## 4. 캐릭터 출전 규칙

- 최대 3명의 캐릭터가 출전한다.
- 각 캐릭터는 한 포지션 또는 조준 방향을 맡는다.
- 기본 구상은 캐릭터 3명 출전 시 A/B/C 포지션에 한 명씩 배치하는 것이다.
- 첫 프로토타입은 기본 캐릭터 1명만 출전한다.
- 기본 캐릭터는 중앙 기준 현재 조준 방향의 120도 범위를 공격한다.
- 캐릭터가 1명 또는 2명만 출전하는 모드가 있는지는 추후 결정한다.
- 캐릭터 세부 기획은 각 캐릭터별로 별도 진행한다.

### 로테이션 조작

유저가 화면을 한 번 터치하면 모든 출전 캐릭터가 시계방향으로 한 포지션씩 이동한다.

예시:

| 터치 전 | 터치 후 |
| --- | --- |
| 1번 캐릭터: A 포지션 | 1번 캐릭터: B 포지션 |
| 2번 캐릭터: B 포지션 | 2번 캐릭터: C 포지션 |
| 3번 캐릭터: C 포지션 | 3번 캐릭터: A 포지션 |

확정 사항:

- 터치는 화면 어디서든 입력된다.
- 로테이션 쿨다운은 없다.
- 로테이션은 아주 빠른 속도로 진행된다.
- 로테이션 중에는 공격할 수 없다.
- 로테이션 중 추가 터치도 무시하지 않고 다음 로테이션으로 이어서 처리한다.
- 다음 로테이션 위치를 미리 보여주지 않는다.

## 5. 전투 흐름

기본 전투 루프:

1. 웨이브 시작
2. 여러 방향에서 적 등장
3. 캐릭터들이 현재 포지션에서 자동 공격
4. 유저는 적 분포와 캐릭터 화력 배치를 보고 터치로 캐릭터 위치를 로테이션
5. 적이 타워에 닿으면 타워 체력 감소
6. 적 처치 등으로 경험치 획득
7. 경험치가 충분하면 업그레이드 선택
8. 웨이브 클리어 후 다음 웨이브 진행

플레이 판단 요소:

- 어느 방향 압박에 어떤 캐릭터를 보낼 것인가
- 강한 캐릭터가 필요한 위치까지 몇 번 터치해야 하는가
- 로테이션 중 짧은 공격 불가 시간을 감수할 만한가
- 캐릭터의 공격 범위가 인접 방향까지 닿는가
- 업그레이드 조합이 로테이션 타이밍을 어떻게 바꾸는가

## 6. 캐릭터 설계 방향

캐릭터는 단순히 피해량만 다른 것이 아니라, 로테이션 판단을 다르게 만들도록 설계한다.

캐릭터 세부 기획은 이 공통 문서에서 한 번에 확정하지 않고, 각 캐릭터별로 별도 진행한다. 이 섹션은 캐릭터별 세부 기획을 만들 때의 공통 방향만 정리한다.

### 캐릭터 역할 예시

| 역할 | 기본 특징 | 로테이션상 의미 |
| --- | --- | --- |
| 단일 저격형 | 강한 적 하나를 빠르게 처치 | 보스/정예가 몰린 방향으로 보내고 싶음 |
| 광역 제압형 | 넓은 각도나 범위 공격 | 잡몹이 몰린 방향에 강함 |
| 지연/군중제어형 | 둔화, 밀침, 속박 | 타워 근처까지 접근한 적을 버티게 함 |
| 지속 피해형 | 독, 화상, 저주 | 오래 머무른 방향에 누적 효율이 좋음 |
| 지원형 | 인접 포지션 방향 버프, 타워 회복, 보호막 | 위치 자체가 파티 운영에 영향을 줌 |

### 공격 범위

- 기본 공격 범위는 캐릭터마다 다를 수 있다.
- 일부 캐릭터는 좁지만 강한 공격을 한다.
- 일부 캐릭터는 약하지만 넓은 각도 또는 인접 방향까지 공격한다.
- 업그레이드로 공격각, 사거리, 관통, 탄 퍼짐을 확장할 수 있다.
- 인접 방향 타격은 이 게임의 주요 성장 체감 요소로 쓸 수 있다.

## 7. 로그라이크 성장 방향

경험치를 모아 레벨업하면 선택지 3개 중 1개를 고르는 구조를 기본으로 한다.

경험치 획득 방식:

- 적 처치 시 경험치를 얻는다.
- 경험치 오브는 자동 흡수된다.
- 레벨업 업그레이드 선택 중에는 전투가 정지된다.

업그레이드 종류:

| 종류 | 예시 |
| --- | --- |
| 캐릭터 단독 강화 | 공격력 증가, 공격 속도 증가, 치명타 확률 증가 |
| 공격 범위 강화 | 공격각 증가, 사거리 증가, 인접 방향 관통 |
| 로테이션 강화 | 이동 후 첫 공격 강화, 로테이션 속도 증가, 로테이션 직전/직후 효과 |
| 포지션 강화 | 특정 포지션에 장판 설치, 포지션별 방어 보너스 |
| 타워 강화 | 타워 체력 증가, 자동 포탑, 보호막 |
| 파티 시너지 | 캐릭터가 특정 순서로 배치되면 효과 발동 |
| 적 대응 강화 | 보스 피해 증가, 빠른 적 둔화, 다수 적 폭발 |

### 로테이션 관련 업그레이드 예시

| 이름 | 효과 |
| --- | --- |
| 선제 타격 | 로테이션 직후 각 캐릭터의 첫 공격 피해 증가 |
| 고속 재정렬 | 로테이션 이동 시간이 더 짧아짐 |
| 교차 엄호 | 로테이션 직전 맡던 방향에 짧은 잔상 공격 |
| 넓은 시야 | 담당 방향 양옆으로 공격 각도 증가 |
| 원형 진형 | 모든 캐릭터 공격 범위가 조금씩 증가 |
| 위기 감지 | 타워에 가까운 적이 있는 방향으로 이동한 캐릭터가 추가 공격 |

## 8. 적 설계 방향

적은 로테이션 판단을 흔드는 방향으로 설계한다.

| 적 타입 | 특징 | 요구 판단 |
| --- | --- | --- |
| 일반병 | 평균 속도와 체력 | 기본 화력 확인 |
| 러너 | 빠르게 타워로 돌진 | 빠른 로테이션 필요 |
| 탱커 | 느리지만 체력 높음 | 강한 캐릭터 배치 필요 |
| 무리형 | 다수로 등장 | 광역 캐릭터 필요 |
| 방패병 | 특정 방향 피해 감소 | 인접 방향 공격/관통 활용 |
| 분열체 | 죽으면 작은 적 생성 | 광역 또는 지속 피해 필요 |
| 정예 | 특정 방향을 압박 | 로테이션 타이밍 압박 |
| 보스 | 여러 방향을 넘나들거나 넓은 압박 생성 | 파티 전체 빌드 검증 |

## 9. 스테이지와 웨이브 구조

초안 구조:

- 1런은 여러 스테이지로 구성한다.
- 각 스테이지는 여러 웨이브로 구성한다.
- 업그레이드는 웨이브 보상이 아니라 경험치 레벨업 보상으로 획득한다.
- 적 등장 방향과 구성은 스테이지의 각 웨이브마다 미리 정해둔다.
- 적 등장 방향은 플레이어에게 미리 예고하지 않는다.
- 일정 웨이브마다 정예 또는 보스가 등장한다.

예시:

| 구간 | 내용 |
| --- | --- |
| 1~2웨이브 | 기본 적과 로테이션 조작 학습 |
| 3웨이브 | 빠른 적 추가 |
| 4웨이브 | 탱커 또는 무리형 적 추가 |
| 5웨이브 | 정예 적 |
| 6~9웨이브 | 업그레이드 조합 검증 |
| 10웨이브 | 보스 |

## 10. MVP 범위

첫 프로토타입에서 확인해야 할 것은 "한 번 터치로 3명의 캐릭터가 회전하는 조작이 재미있는가"다.

최소 구현 범위:

- 중앙 타워 1개
- 사각 전장
- 3개 캐릭터 포지션
- 캐릭터 3명
- 터치 시 시계방향 로테이션
- 웨이브별로 정해진 방향에서 적 스폰
- 적이 중앙으로 이동
- 캐릭터 자동 공격
- 타워 체력
- 경험치와 레벨업
- 웨이브 5개
- 레벨업 시 업그레이드 선택 3개
- 게임 오버와 클리어 결과

MVP에서 미뤄도 되는 것:

- 캐릭터 수집/육성 메타
- 장비
- 복잡한 스킬 컷신
- 많은 적 타입
- 보스 패턴 다변화
- 세밀한 밸런스
- 저장 기능

## 11. UI/UX 초안

모바일 세로 화면 기준으로 설계한다.

- 게임은 모바일 세로형을 기본 화면비로 한다.
- 화면 중앙: 타워와 사각 전장
- 타워 주변: 캐릭터 현재 위치
- 바깥쪽: 적 스폰과 이동 경로
- 상단: 웨이브, 타워 체력, 경험치
- 하단: 캐릭터 상태, 업그레이드 요약
- 전투 중 주 입력: 화면 터치 1종

중요한 시각 정보:

- 현재 각 캐릭터의 포지션
- 현재 기본 캐릭터의 조준 방향
- 타워에 가까운 적
- 캐릭터 공격 범위가 닿는 영역
- 로테이션 중 공격 불가 상태

표시하지 않는 정보:

- 위험 구역 강조
- 다음 로테이션 위치 미리보기
- 다음 적 등장 방향 예고

## 12. 현재 확정된 내용

- 기존 게임에 영향이 가지 않게 새 폴더에서 작업한다.
- 장르는 로그라이크 RPG 타워 디펜스다.
- 게임은 모바일 세로형이다.
- 모바일에서 화면 확대가 되지 않도록 막는다.
- 타워는 맵 한가운데에 있다.
- 적은 사방에서 중앙 타워로 몰려온다.
- 전장은 사각 맵이다.
- 전장을 눈에 보이는 3구역으로 나누지 않는다.
- 캐릭터는 최대 3명 출전 가능하다.
- 첫 프로토타입은 기본 캐릭터 1명만 사용한다.
- 기본 캐릭터는 중앙 기준 120도 공격 범위를 가진다.
- 각 캐릭터는 한 포지션씩 맡는다.
- 캐릭터 세부 기획은 각 캐릭터별로 진행한다.
- 업그레이드에 따라 공격각, 사거리, 인접 방향 타격이 강화될 수 있다.
- 유저가 화면을 한 번 터치하면 각 캐릭터는 시계방향 옆 포지션으로 이동한다.
- 터치 로테이션 쿨다운은 없다.
- 로테이션은 아주 빠르지만, 로테이션 중에는 공격할 수 없다.
- 로테이션 중 추가 터치도 다음 로테이션으로 처리한다.
- 업그레이드는 경험치를 모아 레벨업할 때 획득한다.
- 레벨업 업그레이드 선택 중에는 전투가 정지된다.
- 경험치 오브는 자동 흡수된다.
- 포지션은 실제 위치가 아니라 조준 방향으로 사용한다.
- 포지션에는 기본 공격 방향이나 기본 공격각이 없고, 방향과 각도는 캐릭터별로 정한다.
- 적 등장 방향은 스테이지의 웨이브마다 미리 정하되, 플레이어에게 예고하지 않는다.
- 위험 구역과 다음 로테이션 위치는 표시하지 않는다.
- 앞으로 기획 변경이 생기면 이 문서에 반영한다.

## 13. 아직 정해야 할 내용

### 전투 조작

- 화면 터치 외에 스킬 버튼이 필요한가
- 플레이어가 로테이션 방향을 바꿀 수 있는가, 항상 시계방향인가

### 캐릭터

- 기본 캐릭터 이후 추가 캐릭터 콘셉트
- 캐릭터별 공격 방식과 공격 범위
- 캐릭터별 고유 스킬 존재 여부
- 3명 미만 출전이 가능한지
- 캐릭터 사망 또는 전투 불능이 있는지

### 포지션과 맵

- 적 이동 경로는 직선 방사형인지, 곡선 경로인지
- 캐릭터별 인접 방향 공격은 각도 기반인지, 별도 범위 기반인지

### 로그라이크 성장

- 업그레이드 등급을 둘 것인지
- 캐릭터별 업그레이드와 공용 업그레이드 비율
- 같은 업그레이드 중복 획득 가능 여부
- 실패 후 영구 성장 메타를 넣을지

### 적과 밸런스

- 웨이브마다 적 수를 어떻게 늘릴지
- 보스가 맵 안에서 이동 방향을 바꾸는지
- 타워 체력과 회복 수단
- 난이도 상승 곡선

### UX와 화면

- 공격 범위를 항상 보여줄지, 선택적으로 보여줄지
- 한 손 조작을 기준으로 할지
- 가로 화면 지원 여부

## 14. 다음 작업 제안

우선순위는 아래 순서가 좋다.

1. 기본 캐릭터의 세부 수치와 업그레이드 풀을 다듬는다.
2. 추가 캐릭터의 기본 공격 방향과 공격각을 정한다.
3. 레벨업 업그레이드 선택지의 풀과 등급 구조를 정한다.
4. MVP용 적 타입 3종을 정한다.
5. 첫 프로토타입의 5웨이브 구성을 만든다.
6. 업그레이드 선택지 15~20개를 만든다.
7. 플레이 가능한 HTML/JS 프로토타입으로 검증한다.
